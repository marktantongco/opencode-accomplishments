▸ STEP 1

Three.js inits the scene

In a useEffect hook with empty deps, we create the scene, camera (position.z=5), renderer, and mesh. The mesh is added to the scene. This is the only place Three.js objects are created.

// ── 1. Three.js init: scene, camera, renderer, objects ── const scene = new THREE.Scene(); const camera = new THREE.PerspectiveCamera(60, w/h, 0.1, 1000); camera.position.z = 5; // GSAP will animate this later const renderer = new THREE.WebGLRenderer({ antialias: true, alpha: true }); mountRef.current.appendChild(renderer.domElement); const mesh = new THREE.Mesh(geometry, material); scene.add(mesh);
▸ STEP 2

GSAP owns the camera

Inside gsap.context() (scoped to the mount ref), we tween camera.position.z on scroll using ScrollTrigger. We also start an independent rotation loop on the mesh — both are GSAP-owned, neither touches React.

// ── 2. GSAP init: camera + 3D scroll animations ── const ctx = gsap.context(() => { // Camera dolly on scroll (scrubbed) gsap.to(camera.position, { z: 2, ease: 'none', scrollTrigger: { trigger: mountRef.current, start: 'top top', end: 'bottom bottom', scrub: 1 } }); // Mesh rotation — independent infinite loop gsap.to(mesh.rotation, { y: Math.PI * 2, duration: 8, repeat: -1, ease: 'none' }); }, mountRef); // scoped — ctx.revert() kills both
▸ STEP 3

Framer Motion owns the overlay

The HUD, section titles, status panel, and code toggles are all Framer Motion <motion.div> components. The overlay layer has pointerEvents: 'none' on the container so scroll passes through to the window; individual interactive children re-enable pointer events.

// ── 3. Framer Motion overlay (React) ── <motion.div style={{ position: 'fixed', inset: 0, zIndex: 10, pointerEvents: 'none' }} // never block scroll > <motion.h1 initial={{ opacity: 0, y: 20 }} whileInView={{ opacity: 1, y: 0 }} viewport={{ once: true }} transition={{ duration: 0.6, ease: 'easeOut' }} > Section Title </motion.h1> </motion.div>
▸ STEP 4

Cleanup is non-negotiable

The useEffect return function cancels the RAF, reverts the GSAP context (kills all tweens + ScrollTriggers), disposes Three.js renderer/geometry/material, and removes the canvas from the DOM. Skip any step and you leak memory or break the next route.

// ── 4. Cleanup: kill GSAP, cancel RAF, dispose Three.js ── return () => { cancelAnimationFrame(raf); ctx.revert(); // kills all GSAP tweens + ScrollTriggers renderer.dispose(); geometry.dispose(); material.dispose(); if (renderer.domElement.parentNode) { renderer.domElement.parentNode.removeChild(renderer.domElement); } };
